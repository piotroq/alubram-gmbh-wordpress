## 2. Logorichtlinien

### 2.1 Logo-Anatomie

Das ALUBRAM Logo besteht aus zwei untrennbaren Kernelementen:

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   ═══════════════════  ← Horizontale Streifen (3–5 Linien)       │
│    A L U B R A M       ← Wordmark (Bold-Industrie-Schrift)       │
│   ───────────────────  ← Trennlinie                              │
│   [dunkel] [grau]      ← Farbblock-Element                       │
│                                                                  │
│   ZÄUNE TORE BALKONE CARPORTS  ← Deskriptor (Subline)            │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**Designsprache des Logos:**
Die horizontalen Streifen symbolisieren **Aluminium-Strangpressprofile** — das Kernprodukt des Unternehmens. Die Layering-Struktur vermittelt Präzision, industrielle Stärke und moderne Ästhetik.

```
        ┌─ X ─┬────────────────────────────────┬─ X ─┐
        │     │                                │     │
       X│     │       A L U B R A M            │     │X
```

### 2.5 Logo-Missbräuche (Verboten)

Die folgenden Anwendungen sind **ausdrücklich verboten:**

- ❌ Logo verzerren, strecken oder in den Proportionen verändern
- ❌ Logo rotieren oder schräg stellen
- ❌ Logofarben durch nicht genehmigte Farben ersetzen
- ❌ Dem Logo Schatten, Verläufe oder Glanzeffekte hinzufügen
- ❌ Das Logo auf unruhigen Fotohintergründen ohne Schutzfolie platzieren
- ❌ Die Streifen-Elemente vom Wordmark trennen
- ❌ Den Deskriptor (Subline) eigenständig als Logo verwenden
- ❌ Das Logo in einem Farbraum verwenden, der mit der Markenidentität kollidiert

## 4. Typografie

### 4.1 Typografie-System

ALUBRAM verwendet ausschließlich das **System-Font-Stack** — eine moderne, DSGVO-konforme Lösung ohne externe Font-Dateien, die höchste Performance und native Lesbarkeit gewährleistet.

#### Primärer Schrift-Stack

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
             Oxygen-Sans, Ubuntu, Cantarell, "Helvetica Neue", sans-serif;
```

**Warum System Fonts?**
- ✅ Keine externen Font-Anfragen → volle DSGVO-Konformität (keine Datenübertragung an Google)
- ✅ Maximale Performance (keine Font-Ladezeit, kein FOUT)
- ✅ Native Lesbarkeit auf jedem Gerät (macOS, Windows, Android, iOS)
- ✅ Systemschriften sind perfekt für das Betriebssystem optimiert
- ✅ Geringstes LCP / CLS-Risiko

**System-Fonts nach Betriebssystem:**

| Betriebssystem | Rendert als |
|---------------|-------------|
| macOS / iOS | San Francisco (SF Pro) |
| Windows 10/11 | Segoe UI |
| Android | Roboto |
| Linux (Ubuntu) | Ubuntu |
| Linux (andere) | Cantarell |
| Fallback | Helvetica Neue / Arial |

---

### 4.3 Typografische Regeln

**Abstände:**
- Paragraph-Abstand nach: `1em` (entspricht der Schriftgröße)
- Section-Abstand: `80px` (Desktop), `50px` (Mobile)
- Heading → Body: `24px`

**Zeilenbreite (Lesbarkeit):**
- Optimale Zeilenlänge: `60–75 Zeichen` (entspricht ca. `600–750px` bei 16px)
- CSS: `max-width: 72ch` für Fließtextcontainer

**Buchstabenabstand (Letter Spacing):**
- Normale Fließtext: `0` (Standard)
- Headings H1–H3: `-0.025em` (leicht negativ für Premium-Look)
- OVERLINE/LABELS: `+0.15em` (positiv für Lesbarkeit in Versalien)

**Hervorhebungen:**
- Wichtige Wörter: `font-weight: 700`, **keine Unterstreichung**
- Links: `color: #E21212`, underline bei hover
- Zitate: kursiv + linker Rand-Balken in `#E21212`

## 5. Bildsprache & Fotografie

### 5.1 Fotografiestil

Die ALUBRAM-Bildsprache folgt klaren Prinzipien, die Qualität, Authentizität und Premium-Positionierung kommunizieren.

**Core Attribute:**

```
AUTHENTISCH     Echte Projekte, echte Montagesituationen, keine Stock-Klischees
MATERIALTREU    Aluminium soll glänzen — Metallic-Reflexe, Oberflächentexturen zeigen
KONTEXTUELL     Produkte im eingebauten Zustand (Garten, Haus, Terrasse) abbilden
ATMOSPHÄRISCH   Goldstundenbeleuchtung (kurz nach Sonnenaufgang / vor Sonnenuntergang)
PREMIUM         Hohe Auflösung, professionelle Tiefenschärfe, saubere Komposition
```

---

### 5.2 Fotokategorien

#### Hero-Bilder (Fullscreen)
- **Format:** Querformat 16:9, min. 1920 × 1080 px, ideal 3840 × 2160 px (4K)
- **Inhalt:** Aluminium-Produkt im stimmungsvollen Umfeld (Garten, Einfahrt, Terrasse)
- **Licht:** Abendsonne, Gegenlicht, dramatische Wolken — hoher Dynamikumfang
- **Overlay:** Dark-Gradient von unten `rgba(17,27,45,0.8)` für Textlesbarkeit

#### Produktbilder
- **Format:** Quadratisch 1:1 oder 4:3 für Kacheln; 3:2 für Detailansichten
- **Hintergrund:** Neutral hellgrau `#F2F2F2` oder natürliche Umgebung
- **Aufleuchtung:** Gleichmäßig, keine harten Schatten, Metallic-Glanz erhalten
- **Perspektive:** Frontal + 45°-Ansicht für Raumwirkung

#### Prozess & Team-Bilder
- **Format:** Querformat 3:2
- **Inhalt:** Montage-Situationen, Werkstatt, Beratungsgespräche
- **Stil:** Dokumentarisch, energiegeladen, kompetent wirkend

#### Referenz-Fotografie (Abgeschlossene Projekte)
- **Format:** Mix aus Weitwinkel (Gesamtansicht) und Nahaufnahmen (Details)
- **Qualität:** Immer bei gutem Wetter, gepflegte Umgebung
- **Pflicht:** Vor-/Nach-Vergleiche wo möglich

---

### 5.3 Bildbearbeitung & Filter

**Farbanpassungen für Konsistenz:**
```
Sättigung:    leicht erhöht (+10–15%)  — Aluminium-Blautöne stärken
Kontrast:     moderat erhöht (+5–10%)  — Tiefe und Klarheit
Weißabgleich: leicht warm (ca. 5500K)  — einladende Atmosphäre
Schärfe:      Kanten schärfen (Details bei Metalloberflächen)
Vignette:     sehr subtil             — Fokus auf Produkt lenken
```

**Verboten:**
- ❌ Übersättigte, HDR-übertriebene Bearbeitung
- ❌ Unnatürlich blauer Himmel
- ❌ Billige Stock-Fotos ohne Österreich-Bezug
- ❌ Polnischsprachige Dateinamen oder Alt-Texte (z.B. „Bez-nazwy-1")
- ❌ Bilder unter 800 px Breite für Web-Veröffentlichung

| „Unbegrenzte Farbauswahl aus der RAL-Palette" | Individualität betonen |

| „Wir kommen zu Ihnen — österreichweit" | Lokale Präsenz |
| „Jetzt unverbindlich anfragen" | Niedrigschwelliger CTA |

**DON'T — Diese Formulierungen vermeiden:**

| Schlecht ❌ | Weil |
|------------|------|
| „Wir versuchen..." | Keine Unverbindlichkeit |
| „Möglicherweise können wir..." | Unsicherheit kommunizieren |
| „billig" / „günstig" | Premium-Positionierung untergraben |
| Passivkonstruktionen (z.B. „Es wird gefertigt...") | Zu unpersönlich |
| Übermäßige Superlative ohne Beleg | Unglaubwürdig |
| Englische Buzzwords ohne Erklärung | Kundengruppe ausschließen |
| Polnische Formulierungen im DE-Text | Marktpositionierung verwässern |

## 12. Do's & Don'ts

### 12.1 Logo

| ✅ DO | ❌ DON'T |
|-------|---------|
| Logo mit ausreichend Weißraum (Schutzzone) einsetzen | Logo unter Mindestgröße verkleinern |
| Genehmigte Farbvarianten verwenden | Logo verzerren, strecken oder rotieren |
| Logo auf neutralem oder Marken-Hintergrund platzieren | Schatten oder Glanzeffekte hinzufügen |
| SVG-Format für digitale Medien nutzen | Niedrigauflösendes Logo für Druck verwenden |
| Logo-Elemente zusammen verwenden | Streifen und Wordmark trennen |

### 12.2 Farben

| ✅ DO | ❌ DON'T |
|-------|---------|
| Primärfarben nach Hierarchie einsetzen (#E21212 für CTAs) | Spontane Farbzugaben ohne Systemzugehörigkeit |
| WCAG AA-Kontrast (4.5:1) immer einhalten | Rot auf Rot oder Navy auf Navy kombinieren |
| CSS Custom Properties für Konsistenz nutzen | Markenfarben mit Schwarz 100% mischen |
| Aluminium-Metalltöne als Ergänzungspalette | Bunte Komplementärfarben (Grün, Orange, Lila) |

### 12.3 Typografie

| ✅ DO | ❌ DON'T |
|-------|---------|
| System Font Stack für DSGVO-Konformität | Externe Font-Dienste ohne Einwilligung (Google Fonts CDN) |
| Klare H1→H6-Hierarchie pro Seite | Mehrere H1 auf einer Seite |
| Zeilenlänge max. 72–75 Zeichen | Zeilen > 100 Zeichen (unlesbar) |
| Konsequent „Sie"-Form verwenden | Mischung Sie/du Ansprache |
| Deutsche Umlaute (ä, ö, ü, ß) korrekt verwenden | Umlaute ersetzen (ae/oe/ue) |

### 12.4 Fotografie

| ✅ DO | ❌ DON'T |
|-------|---------|
| Eigene Referenzfotos mit gutem Licht | Billige Stockfotos ohne Österreich-Bezug |
| Beschreibende Alt-Texte auf Deutsch | Polnische Platzhalter-Dateinamen |
| .avif/.webp Formate mit JPEG-Fallback | Nur .png für Produktfotos (zu groß) |
| Bilder unter Goldstundenlicht aufnehmen | Überbearbeitete HDR-Effekte |

### 12.5 Copywriting

| ✅ DO | ❌ DON'T |
|-------|---------|
| Konkrete Zahlen nennen (3–6 Wochen, 15 Jahre) | Vage Versprechen ohne Belege |
| USPs in jede Kommunikation einweben | Allgemeine Werbefloskeln |
| Positives Priming: Vorteile zuerst | Lange Einleitungen ohne Kernaussage |
| Regionalen Bezug herstellen (Wien, NÖ, DACH) | Zu allgemein (kein lokaler Bezug) |
